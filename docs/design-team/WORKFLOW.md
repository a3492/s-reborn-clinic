# 콘텐츠 디자인 워크플로우 (Pre / Mid / Post)

> 모든 콘텐츠는 이 3단계 워크플로우를 거쳐야 합니다.
> Cursor AI는 콘텐츠 관련 요청 시 해당 단계를 명시하며 지원합니다.

---

## 📋 Phase 1 — 제작 전 (Pre-Production)

새 콘텐츠 제작 요청이 들어오면 즉시 아래를 확인하고 제공:

| 항목 | 내용 |
|------|------|
| 썸네일 가이드 | 사이즈(1200×800px), 구도, 텍스트 오버레이 규칙 |
| 제목 감성 검토 | 차갑지 않은가? 공감되는가? 클릭하고 싶은가? |
| 카테고리/태그 | 디자인 레이블 매핑 (기존 시스템과 일관성 확인) |
| 인포그래픽 필요성 | 시술 설명인가? → 일러스트 필요 여부 판단 |
| CTA 사전 결정 | 버튼 문구, 위치, 연결 페이지 |

**체크리스트 템플릿**:
```
[ ] 썸네일 이미지 준비 (3:2 비율, 1200×800px 이상)
[ ] 제목 감성 검토 완료
[ ] 카테고리/태그 배정 완료
[ ] OG 이미지 생성 계획 수립
[ ] CTA 문구 및 위치 결정
[ ] 관련 인포그래픽 필요 여부 결정
```

---

## ✏️ Phase 2 — 제작 중 (Mid-Production)

콘텐츠 작성 과정에서 실시간 지원:

| 항목 | 가이드 |
|------|--------|
| 소제목(H2/H3) | `--font-heading` 스타일, `--color-deep` 색상 적용 |
| 이미지 삽입 | 본문 600~800자마다 1회, 캡션은 `--text-sm` + `--color-accent` |
| 강조 텍스트 | 볼드는 단락당 최대 1~2개, 컬러 강조는 `--color-accent` |
| 목록/표 | 불릿 리스트는 최대 5개 항목, 표는 배경 `--color-highlight` |
| 본문 길이 | 800~1200자 권장, 2000자 초과 시 목차(TOC) 추가 |

**실시간 확인 포인트**:
```
[ ] H2/H3 소제목이 너무 많지 않은가? (500자당 1개 기준)
[ ] 이미지가 텍스트 흐름을 방해하지 않는가?
[ ] 강조 표시가 과도하지 않은가?
[ ] 의학 용어가 공감 언어로 변환되었는가?
[ ] 단락 간 여백이 충분한가?
```

## ✅ Phase 3 — 제작 후 (Post-Production)

퍼블리싱 전 최종 점검:

| 항목 | 확인 방법 |
|------|----------|
| 모바일 렌더링 | 375px 뷰포트 시뮬레이션 |
| 다크모드 | `data-theme="dark"` 전환 후 가독성 확인 |
| OG 이미지 | 소셜 공유 썸네일 1200×630px 생성 확인 |
| 내부 링크 | 링크 스타일 `--color-accent` 일관성 |
| 성능 | 이미지 WebP 변환, 100KB 이하 권장 |
| 타이포그래피 | 모바일에서 `--text-base` 16px 이상 유지 |

**최종 게시 체크리스트**:
```
[ ] 모바일(375px) 렌더링 확인
[ ] 다크모드 가독성 확인
[ ] OG 이미지 생성 완료
[ ] 내부 링크 디자인 일관성 확인
[ ] 이미지 최적화 완료 (WebP, 압축)
[ ] 메타 제목/설명 감성 검토 완료
[ ] 카테고리/태그 최종 확인
[ ] 관련 글 섹션 연결 확인
```
