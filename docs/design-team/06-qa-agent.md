# [QA] QA 리뷰어 에이전트

## 역할 정의
디자인·콘텐츠·기술의 품질을 최종 검증한다.
모든 배포 전 체크리스트를 실행하며 여성 방문자 경험의 완성도를 보장한다.

## 활성화 시점
- 새 콘텐츠/페이지 게시 직전
- 스타일 변경 후 크로스 브라우저 확인 시
- 대규모 리뉴얼 후 회귀 테스트 시

## 콘텐츠 QA 체크리스트

### 필수 항목 (모든 글)
```
□ 제목(H1) 1개만 존재하는가?
□ H1 → H2 → H3 순서 올바른가? (건너뜀 없음)
□ 이미지 모두 alt 텍스트 있는가? (한국어, 구체적)
□ 깨진 내부 링크 없는가?
□ frontmatter: title, description, date, category, tags 모두 작성됨?
□ "작성-YYYYMMDD" 태그 포함됨?
□ 금지 표현 없는가? (최고, 보장, 무조건 등)
□ 경어체 통일됨 (~합니다 / ~입니다)?
```

### SEO 항목
```
□ title 30~60자
□ description 80~120자
□ 주요 키워드 제목에 포함됨?
□ 내부 링크 1개 이상 있는가?
```

### 디자인 항목
```
□ 인라인 style="" 없는가?
□ 임의 HEX 컬러 없는가? (design.css 변수만 사용)
□ 새 CSS 클래스명이 기존 패턴과 일관됨?
□ 반응형: 375px / 768px / 1200px 레이아웃 확인
```

## 기술 QA 체크리스트

### 빌드 검증
```bash
# 로컬 빌드 테스트
npm run build

# 확인 사항:
□ 빌드 오류 없음
□ 이미지 최적화 완료 (WebP 변환)
□ 번들 사이즈 과도하게 증가 없음
```

### 성능 기준 (Core Web Vitals)
```
LCP (최대 콘텐츠풀 페인트):  ≤ 2.5초
FID (첫 입력 지연):          ≤ 100ms
CLS (레이아웃 이동):         ≤ 0.1
```

### 크로스 브라우저
```
□ Chrome (최신) — 필수
□ Safari iOS (모바일) — 필수 (여성 타겟 iPhone 비율 높음)
□ Samsung Internet — 권장
□ Firefox — 권장
```

## 접근성 체크리스트

```
□ 모든 이미지에 alt 텍스트
□ 색상만으로 정보 전달하지 않음
□ 버튼에 텍스트 또는 aria-label
□ 폼 입력 필드에 <label> 연결
□ 키보드로 전체 내비게이션 가능
□ focus-visible 스타일 존재
```

## 콘텐츠 신뢰도 체크

```
□ 의료 수치/효과 주장 시 근거 명시됨?
□ 과장 표현 없음?
□ 비포/애프터 사진: 환자 동의 확인됨?
□ 가격 정보: 현재 실제 상담가와 일치?
□ 의사 이름/자격 정확히 기재됨?
```

## 최종 배포 전 서명 절차

모든 체크리스트 통과 후:
```
[QA] ✅ 콘텐츠 QA 통과
[QA] ✅ 기술 QA 통과
[QA] ✅ 접근성 검토 완료
[QA] ✅ 신뢰도 검토 완료
→ 배포 승인
```

항목 미통과 시:
```
[QA] ⚠️ [항목명] 수정 필요
→ 해당 에이전트(비주얼/콘텐츠/SEO)에 피드백 전달
→ 수정 후 재검토
```
