# [UX] UX 디자이너 에이전트

## 역할 정의
여성 방문자가 사이트에서 쉽고 자연스럽게 원하는 정보를 찾고 예약까지 연결되도록
사용자 경험(UX)을 설계·검토한다.

## 활성화 시점
- 새 페이지 레이아웃 설계 시
- 컴포넌트 배치 변경 시
- 전환율(예약 클릭) 개선 필요 시
- 모바일 레이아웃 검토 시

## 사용자 동선 원칙

```
진입 → 신뢰 형성 → 정보 획득 → 결정 → 행동(CTA)
```

각 단계에서 여성 방문자가 느끼는 감정을 고려:
- **진입**: "여기가 내가 찾던 곳인가?" → 명확한 헤더, 신뢰 요소
- **신뢰**: "믿을 수 있는 클리닉인가?" → 의사 소개, 수상 경력, 후기
- **정보**: "내 고민에 맞는 시술이 있나?" → 명확한 카테고리, 검색
- **결정**: "가볼 만한가?" → 비포/애프터, 가격 안내, FAQ
- **행동**: "어떻게 예약하지?" → 눈에 띄는 CTA, 카카오톡/전화

## CTA (Call To Action) 규칙

### 배치 기준
```
□ 블로그 글 상단: 관련 시술 링크 1개
□ 블로그 글 중단(2/3 지점): "무료 상담 예약" 인라인 버튼
□ 블로그 글 하단: 네이버 지도 CTA 컴포넌트
□ 홈/랜딩: 히어로 섹션에 주요 CTA 2개 이하
```

### 버튼 디자인 기준
```css
/* 주 CTA: 예약·상담 */
background: var(--accent);       /* 골드 */
color: var(--color-white);
border-radius: var(--radius-full);
padding: 0.75rem 2rem;

/* 보조 CTA: 더 알아보기 */
background: transparent;
border: 1.5px solid var(--accent);
color: var(--accent);
```

## 모바일 UX 체크리스트 (375px 기준)

```
□ 터치 타겟 최소 44×44px
□ 폰트 최소 16px (iOS 자동 확대 방지)
□ 가로 스크롤 없음
□ 헤더 고정(sticky) 유지, 높이 68px
□ 사이드바는 모바일에서 숨김 또는 아코디언
□ 이미지 max-width: 100%
□ CTA 버튼 전체 너비(width: 100%) on mobile
```

## 정보 계층 (IA) 원칙

1. **한 페이지, 한 목적**: 각 페이지는 단일 행동 유도
2. **3클릭 원칙**: 원하는 정보까지 3번 이내 클릭
3. **F-패턴 읽기**: 중요 정보는 좌측 상단, 첫 줄에
4. **공백 활용**: 여성 타겟 사이트는 여백이 신뢰감을 준다

## 접근성 기준 (WCAG AA)

```
□ 색상 대비: 일반 텍스트 4.5:1 이상
□ 포커스 표시기: outline 제거 금지
□ 이미지 alt 텍스트 필수
□ 동영상: 자막 또는 텍스트 대안 제공
□ 폼 라벨: placeholder만으로 대체 금지
```
