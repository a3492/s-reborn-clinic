## 2026-04-09

### [FIX-01] 숫자 범위 `~` 취소선 오파싱

**증상**
`4~6주부터 본격화되어 3~6개월` 같이 숫자 사이 물결표를 쓴 본문에서
`6주부터 본격화되어 3` 부분이 `<del>` 취소선으로 렌더링됨.

**원인**
`remark-gfm` 라이브러리의 단일 물결표(`~text~`) 취소선 파싱 기능.
한국어 범위 표기 `A~B`에서 양쪽 `~` 사이 텍스트를 취소선으로 오파싱.

**영향 범위**
`src/content/` 하위 **약 180개 파일** 전체.

**수정 방법**
`astro.config.mjs`에 `remarkGfm({ singleTilde: false })` 옵션 적용.

```js
// astro.config.mjs
markdown: {
  gfm: false,
  remarkPlugins: [[remarkGfm, { singleTilde: false }]],
},
```

**적용 범위**
전역 설정이므로 **기존 + 신규 모든 `.md` / `.mdx` 파일**에 자동 적용.

**커밋**: `fix: remark-gfm singleTilde false — 숫자 범위 물결표 취소선 오류 수정`

### [FIX-02] 본문 `---` 수평선(`<hr>`) 렌더링

**증상**
FAQ·프로토콜 등 콘텐츠 파일 본문에 `---`(단독 줄)이 가로 구분선으로 렌더링되어
취소선처럼 시각적으로 방해됨.

**원인**
Markdown에서 본문 단독 `---`는 `<hr>` 태그로 변환됨.
프론트매터 외 본문에서 구분선 목적으로 남용된 패턴.

**영향 범위**
`src/content/` 하위 **139개 파일** (시술 콘텐츠 + 블로그 전 카테고리).

**수정 방법**
CSS로 본문 영역 `<hr>` 숨김 처리.

```css
/* ProcedurePage.astro */
.proc-article-body :global(hr) { display: none; }

/* global.css */
.blog-post-prose hr { display: none; }
```

**커밋**: `style: 본문 hr 구분선 숨김 (시술 페이지 + 블로그)`

### [FIX-03] `*` 별표 오파싱 조사 결과

**증상**
일부 페이지에서 `*` 문자가 렌더링에 영향을 줄 수 있다는 우려 제기.

**조사 결과**
- 현재 콘텐츠 파일 전수 검토 결과, `**볼드**` 패턴은 모두 정상 짝 일치.
- `*` 단독 오파싱 실사례 없음 확인.
- 터미널 인코딩 깨짐으로 인한 오탐이었음.

**예방 조치**
- `.cursor/rules/content-markdown-safety.mdc` Cursor Rule 생성.
- 향후 `*`를 일반 문자로 쓸 경우 `\*`로 이스케이프 규칙 명문화.

---

## 향후 콘텐츠 작성 주의사항

| 문자 | 잘못된 사용 | 올바른 사용 |
|------|------------|------------|
| `~` | `4~6주` (범위 표기) | `4–6주` (en-dash) 또는 현재 설정으로 안전 |
| `*` | `단어*` (리터럴) | `단어\*` (이스케이프) |
| `**...*` | 볼드 시작, 이탤릭 끝 | `**...**` 짝 일치 |
| `---` | 본문 구분선 | 빈 줄로 섹션 구분 |

> 상세 규칙: `.cursor/rules/content-markdown-safety.mdc`
